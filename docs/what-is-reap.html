Reap: One-command Pear release automation

This project is a Node.js CLI that automates the full Pear app release ceremony: build, stage, provision, multisig signing (or solo mode), config inference, key management, checkpoint/resume, and optional seeding.

Entrypoint
reap release
Config
Single file: .reap.json
Modes
Solo or Multisig
UX
Short, colored, spinner-based output

Architecture

Release Flow

Checkpoints are written during the flow so reap release --resume can continue from the last safe step.

Key idea

Reap removes ceremony. It infers what it can (project/package paths, build inputs, artifacts in out/, Forge signing hints, existing links), warns when something is risky, and only blocks on truly invalid release states.

  • reap configure: interactive bootstrap that fills one config file.
  • reap keys ...: generate/import/rotate/revoke/doctor signer keys.
  • reap signers ...: manage signer roster and quorum.
  • reap release --solo: lone-developer path without multisig ceremony.
